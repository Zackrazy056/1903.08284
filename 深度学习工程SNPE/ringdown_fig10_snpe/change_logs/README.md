# Change Log Policy

- Purpose: keep an auditable record of every production-impacting edit and run.
- One file per session/day: `YYYY-MM-DD_<topic>.md`.
- Required sections for each entry:
  1. Scope (files/functions changed)
  2. Action (exact code/config operation)
  3. Rationale (why this change is needed)
  4. Validation (command/run and observed result)
  5. Risk/Follow-up
- Keep entries append-only; do not rewrite past records.
